# 230MB 仓库推送失败（HTTP 302 / 连接断开）的解决办法

## 方法一：改用 SSH 推送（推荐）

HTTPS 推送大仓库容易超时或 302，SSH 通常更稳。

### 1. 检查是否已有 SSH 公钥
在终端执行：
```bash
cat ~/.ssh/id_rsa.pub
```
或
```bash
cat ~/.ssh/id_ed25519.pub
```
若能看到一长串以 `ssh-rsa` 或 `ssh-ed25519` 开头的内容，说明已有公钥。

### 2. 把公钥加到 GitHub
- 复制上面命令输出的**整行**（含 ssh-rsa 和末尾邮箱）
- 打开 https://github.com/settings/keys
- 点 **New SSH key**，Title 填 `Mac`，Key 里粘贴公钥，保存

### 3. 若出现 Connection reset / port 22：改用 443 端口
网络封了 22 端口时，在终端执行（只需做一次）：
```bash
mkdir -p ~/.ssh
touch ~/.ssh/config
chmod 600 ~/.ssh/config
```
然后打开 `~/.ssh/config`（用文本编辑或 `open -e ~/.ssh/config`），**加入或合并**下面几行后保存：
```
Host github.com
  Hostname ssh.github.com
  Port 443
  User git
```
保存后再执行下面的「用 SSH 地址推送」。

### 4. 用 SSH 地址推送
在项目目录执行：
```bash
cd "/Users/91002302/Desktop/摄影作品集网站"
git remote set-url origin git@github.com:zhangchunyi-zcy/yi-photo.git
git push -u origin main
```
按提示操作（首次可能询问是否信任 host，输入 `yes`）。

## 方法二：继续用 HTTPS，只加长超时

若暂时不想用 SSH，可先试加大超时再推送：
```bash
cd "/Users/91002302/Desktop/摄影作品集网站"
git config http.postBuffer 524288000
git config http.lowSpeedLimit 0
git config http.lowSpeedTime 999999
git push -u origin main
```

## 方法三：先不推照片，只推代码（站点能访问但图片需另存）

若 SSH 和 HTTPS 都失败，可先让 Vercel 能部署出「框架」站点，照片之后再补或放外链。

1. **把照片从本次提交里拿掉（只改当前这次提交）：**
   ```bash
   cd "/Users/91002302/Desktop/摄影作品集网站"
   echo "photos/" >> .gitignore
   git rm -r --cached "photos/"
   git commit --amend -m "Initial: 摄影作品集网站（不含照片）"
   git push -u origin main
   ```
2. 推送成功后到 Vercel 导入 **yi-photo** 部署，先拿到域名。
3. 照片可后续用 **Git LFS** 或图床外链再补（需改 `data.js` 里的路径）。

**注意：** 用方法三的话，线上站点暂时没有图片，只适合先拿到域名、再慢慢补图。

---

建议优先试 **方法一（SSH）**，成功率最高。
