# 在别的电脑上部署（无需 Cursor）

在任意一台电脑（Windows / Mac / 笔记本）上，按下面做即可拿到在线分享链接。不需要 Cursor、Git 或 GitHub。

---

## 一、把项目拷到那台电脑

任选一种方式，把整个 **「摄影作品集网站」文件夹** 复制过去：

- U 盘拷贝
- 网盘（百度网盘、阿里云盘、iCloud 等）上传后在那台电脑下载
- 微信/QQ 发压缩包（先打成 zip 再发，对方解压后得到整个文件夹）

确保那台电脑上的文件夹里包含：`index.html`、`contact.html`、`css/`、`js/`、`photos/` 等，结构和你现在这份一致。

---

## 二、在那台电脑上安装 Node.js（若未安装）

1. 打开浏览器访问：**https://nodejs.org**
2. 下载并安装 **LTS 版本**（推荐）。
3. 安装完成后，**关闭并重新打开**终端（或命令提示符），再继续下一步。

---

## 三、用终端进入项目文件夹

- **Mac / Linux**：打开「终端」，执行（路径改成你实际放项目的路径）：
  ```bash
  cd /路径/到/摄影作品集网站
  ```
  例如放在桌面：
  ```bash
  cd ~/Desktop/摄影作品集网站
  ```

- **Windows**：打开「命令提示符」或「PowerShell」，执行（路径改成你实际路径）：
  ```cmd
  cd C:\Users\你的用户名\Desktop\摄影作品集网站
  ```
  若路径有空格或中文，用英文双引号包起来，例如：
  ```cmd
  cd "C:\Users\你的用户名\Desktop\摄影作品集网站"
  ```

---

## 四、登录 Vercel 并部署

在同一终端里依次执行：

```bash
npx vercel login
```

按提示用浏览器或邮箱登录 Vercel（只需登录一次）。

登录成功后执行：

```bash
npx vercel --prod
```

- 第一次会问项目名、是否关联 Git 等，一般直接**回车用默认**即可。
- 部署完成后，终端最后会给出一个 **Production** 链接，例如：  
  `https://摄影作品集网站-xxxx.vercel.app`  
  这就是你的**在线分享链接**，用浏览器打开或发给别人即可。

---

## 五、之后在那台电脑上更新站点

若你在那台电脑上改了项目里的代码或图片，想更新线上版本，在同一项目文件夹里再执行一次：

```bash
npx vercel --prod
```

即可更新刚才那个链接里的内容。

---

**总结：** 在那台电脑上只需要：① 项目文件夹 ② Node.js ③ 终端里执行 `npx vercel login` 和 `npx vercel --prod`，不依赖 Cursor、Git 或 GitHub。

---

## 为什么这份说明没用 GitHub？可以用 GitHub 吗？

**可以，而且用 GitHub 通常更好。** 当前这份「不用 GitHub」的流程，是因为在你之前那台电脑上往 GitHub 推送一直失败（HTTP 302、连接断开），所以改成本地直接 `npx vercel --prod` 部署，不依赖推送成功。

### 用 GitHub + Vercel 的好处

| 方式 | 不用 GitHub（当前说明） | 用 GitHub + Vercel |
|------|-------------------------|---------------------|
| **更新网站** | 每次改完都要在那台电脑上再跑一次 `npx vercel --prod` | 改完代码后 `git push`，Vercel 会**自动重新部署**，不用再跑命令 |
| **在别的电脑改项目** | 要重新拷整个文件夹，再跑 `npx vercel --prod` | 在那台电脑 `git clone` 仓库，改完 `git push` 即可，线上自动更新 |
| **备份与历史** | 只有本地一份，误删难恢复 | 代码在 GitHub 上，有提交历史，可随时回滚 |
| **协作** | 别人要你发压缩包或网盘 | 发一个仓库链接，对方 clone 即可参与或复用 |

### 若你想用 GitHub，可以这样接上

1. **在那台「别的电脑」上**（或网络更好的环境）：  
   安装 Git，把项目文件夹初始化为仓库，推送到 GitHub（若照片很多，可先不推 `photos/`，或改用 Git LFS）。  
   具体步骤见项目里的 **DEPLOY.md** 和 **GitHub推送认证说明.md**。

2. **在 Vercel 网页**：  
   用 GitHub 登录 → Add New → Project → 选择你的仓库（如 `yi-photo`）→ Deploy。  
   之后每次在任意电脑上 `git push` 到该仓库，Vercel 都会自动部署，无需再跑 `npx vercel --prod`。

3. **在别的电脑上改项目并更新网站**：  
   `git clone https://github.com/你的用户名/yi-photo.git` → 改代码 → `git add .`、`git commit`、`git push`，等一两分钟线上就更新了。

**结论：** 能用 GitHub 就尽量用，方便自动部署和协作；当前文档不用 GitHub，是为了在推送失败时也能先拿到在线链接。等你在别的电脑或别的网络下能稳定推送到 GitHub 时，按 DEPLOY.md 接上即可。
